# 研究日记 V0.1
## 项目名称：基于时空索引的自动驾驶出租车调度系统
## 实验周期：第一阶段迭代（基础功能实现 + 系统稳定化）
## 核心目标：完成五大模块的实现，修复系统异常，实现稳定调度与真实指标输出

---

## 一、整体进展概述
本阶段已**完整实现课程要求的全部 5 大核心模块**，搭建了一套可运行、可观测、可优化的自动驾驶出租车调度仿真系统。
系统基于 SUMO 仿真环境 + SQLite 空间索引，支持 300 辆出租车、2000 辆背景车、1500 单高峰订单的完整调度闭环。
通过不断调参与修复机制，最终实现 **65.13% 订单完成率**，系统稳定运行 3600 秒。

---

## 二、各模块实现内容与研究工作记录

### 0. 主循环引擎模块（Main Engine）
**已实现功能：**
- 完成 SUMO 仿真主循环驱动，按步（Step）推进整个系统运行。
- 实现**车辆无限行驶机制**：对所有车辆进行路线续接，避免车辆到达终点后消失。
- 实现**车辆复活机制**：针对拥堵/碰撞导致车辆消失的问题，自动复活车辆，保证系统始终保持 **300 辆出租车** 稳定在线。
- 统一调度流程：按顺序执行位置更新、订单处理、派单、车辆状态更新、重平衡。

**遇到问题与解决：**
- 车辆因拥堵大量消失 → 增加复活逻辑，保证车队规模稳定。
- 仿真中途崩溃 → 优化路线续接时机，提升系统稳定性。

---

### 1. 空间索引设计模块（Index Design）
**已实现功能：**
- 基于 SQLite 实现 **R-Tree 空间索引**，用于存储出租车实时 (x, y) 位置。
- 每一步通过 TraCI 采集车辆位置，并同步更新数据库与索引。
- 为后续 KNN 快速查询提供底层支持，大幅提升大规模车辆检索效率。

**本阶段贡献：**
完成了调度系统最核心的**时空数据结构**，满足“高效管理移动车辆”的核心要求。

---

### 2. 订单生成与处理模块（Request Handling）
**已实现功能：**
- 在 3600 秒内随机生成 **1500 个乘客请求**，符合高峰需求分布。
- 基于 R-Tree 索引实现 **KNN 最近邻查询**，快速查找乘客附近空闲出租车。
- 构建订单生命周期：未派单、已分配、已完成、已取消。

**本阶段贡献：**
实现了完整的订单输入与最近车辆检索，是调度系统的需求入口。

---

### 3. 智能派单与路由模块（Dispatch Logic）
**已实现功能：**
- 实现 Python 调度控制器，完成订单与车辆的匹配。
- 通过 TraCI 的 `setRoute` 实现车辆路径规划与导航。
- **关键优化：限制 Rebalance 仅使用 20% 空闲车**，保证大部分车辆优先服务订单。
- 修复虚假派单问题：拒绝远距离无效派车，只分配可真实到达的车辆。

**遇到问题与解决：**
- 早期接驾时间极短（虚假指标）→ 修复为真实行驶时间。
- 接驾时间过长（250s+）→ 通过限制跨区域派单、优化路线，使时间回归真实合理。

**本阶段贡献：**
实现真正可用的派单逻辑，系统从“虚假好看”变为“真实可运行”。

---

### 4. 车辆可用性与状态管理模块（Availability Management）
**已实现功能：**
- 实现车辆三状态流转：
  `IDLE` → `PICKUP` → `OCCUPIED`
- 使用**数据库原子事务**保证状态安全，避免一车多派、数据错乱。
- 同步 SUMO 车辆颜色：绿色空闲、橙色接客、红色载客。
- 监听车辆到达事件，自动切换状态，形成闭环。

**本阶段贡献：**
保证系统状态一致性，是调度稳定运行的基础。

---

### 5. 空车重平衡模块（Rebalancing）
**已实现功能：**
- 支持 **17 个区域的跨区域运力调度**。
- 周期性统计各区域供需关系，将空车从车多区域调到订单多区域。
- 严格限制重平衡车辆比例（20%），避免抢走全部空车。

**遇到问题与解决：**
- 重平衡抢走全部车辆 → 加限流，保证订单优先。
- 车辆被派到过远区域 → 优化区域匹配逻辑。

**本阶段贡献：**
实现全局运力优化，降低等待时间与空载率，符合真实网约车运营逻辑。

---

## 三、本阶段关键优化与成果

### 1. 限制 Rebalance 比例（20%）
- 解决：系统崩溃、路网瘫痪、订单无车可接。
- 效果：系统稳定，订单完成率从 **3% → 65%**。

### 2. 修复虚假送达时间
- 问题：早期等待时间极短是“假数据”，车辆并未真正到达。
- 修复：开启真实位置校验。
- 效果：数据从虚假变真实，指标具备科研参考价值。

### 3. 车辆复活与保活机制
- 问题：拥堵导致车辆消失，只剩 150 台车。
- 修复：自动复活，保持 300 台稳定运行。
- 效果：系统运力充足，完成率大幅提升。

### 4. 最终稳定指标
- 订单完成率：**65.13%（977/1500）**
- 平均等待接单时间：96.25s
- 平均接驾时间：152.99s
- 空驶率：30.58%
- 空闲车辆比例：12%
- 系统：稳定跑完 3600 秒
